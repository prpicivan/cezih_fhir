CEZIH Integracija (G9 Middleware)

Dobrodošli u dokumentaciju za CEZIH FHIR Bridge. Vaša G9 aplikacija ovdje komunicira putem jednostavnih REST API poziva. Middleware se brine o izgradnji FHIR HL7 STU3 paketa, kanonizaciji i potpisivanju.

1. Pacijenti

Dohvat pacijenata iz CEZIH-a

GET/api/patient/search/{mbo}
Nema JSON body-ja

2. Posjete (Encounters)

Otvaranje (TC 12) i Zatvaranje (TC 14)

Važno: Klasa posjete mora odgovarati tipu dokumenta koji ćete kasnije slati. Za ambulantne nalaze koristite AMB, za bolničke (otpusna pisma) koristite IMP.

POST/api/visit/create
{ "patientMbo": "999999423", "class": "AMB", "startDate": "2026-03-15T08:00:00Z" }

3. Zdravstveni Slučajevi (Conditions)

Kako funkcioniraju Slučajevi?

Slučaj (Condition) ne predstavlja samo jedan dolazak, već cjelokupni tijek liječenja jedne bolesti (npr. Liječenje hipertenzije). CEZIH strogo prati "Stroj stanja" (State Machine) za svaki slučaj.

Zlatno pravilo povezivanja:

Ako pacijent već ima otvoren slučaj na CEZIH-u za određenu MKB-10 dijagnozu (npr. "I10 - Hipertenzija"), NE SMIJETE otvarati novi slučaj. Morate dohvatiti taj postojeći slučaj i njegov caseId proslijediti prilikom slanja nalaza.

Ispravan flow za kronične bolesti:
1. Dohvati slučajeve (GET) -> 2. Odaberi postojeći caseId -> 3. Otvori novu Posjetu (TC 12) -> 4. Pošalji Nalaz (TC 18) vezan uz tu posjetu i stari caseId.

Životni ciklus Slučaja:

1. AKTIVAN (Nova bolest)

Stvara se preko `/api/case/create`.

↓ 2.3 Remisija ↓
2. REMISIJA (Bolest miruje)

Slučaj je i dalje otvoren, ali pacijent nema simptome.

↙ 2.5 Relaps ↙
↘ 2.4 Zatvaranje ↘
RELAPS

Vraća u AKTIVAN

ZATVOREN

Izliječen / Gotovo

Šifrarnik Akcija (TC 17):

  • 2.4 - Zatvaranje slučaja (Pacijent izliječen ili je obrada završena).
  • 2.3 - Proglašenje remisije (Bolest miruje).
  • 2.5 - Relaps/Recidiv (Povratak bolesti iz stanja remisije).
  • 2.8 - Ponovno otvaranje (Vraćanje greškom zatvorenog slučaja u Aktivno stanje).

Otvaranje novog slučaja (TC 16)

Pozivate samo ako pacijent nema već aktivan slučaj za ovu dijagnozu na CEZIH-u.

GET/api/case/remote/{mbo} (Dohvat)
POST/api/case/create (Kreiranje)

Request (Kreiranje slučaja)

{ "patientMbo": "999999423", "diagnosisCode": "I10", "title": "Hipertenzija" }

Ažuriranje statusa slučaja (TC 17)

Mijenja status kroz stroj stanja. Prosljeđujete ID slučaja i šifru akcije.

POST /api/case/{id}/action

Request (Zatvaranje slučaja)

{ "patientMbo": "999999423", "action": "2.4", "reason": "Terapija uspješno završena." }

4. Klinički Nalazi (Dokumenti)

Tipovi dokumenata i pravila

Prilikom slanja dokumenta na /api/document/send, morate točno definirati parametar type. Svaki tip dokumenta ima svoja pravila i zahtijeva specifičnu Klasu posjete (Encounter Class).

Šifra (type) Naziv dokumenta Klasa Posjete (TC12) Opis / Korištenje
011 Izvješće nakon pregleda u ambulanti AMB (Ambulatory) Standardni pregled kod liječnika opće prakse ili privatnika.
012 Specijalistički nalaz AMB (Ambulatory) Nalaz liječnika specijaliste nakon uputnice ili konzultacije.
013 Otpusno pismo IMP (Inpatient) Izdaje se isključivo pacijentima nakon stacionarnog liječenja (ležanja u ustanovi).

⚠️ CEZIH će ODBITI dokument ako pošaljete Otpusno pismo (013), a posjeta na koju se vežete je označena kao AMB (Ambulantna)!

Pretraživanje nalaza (TC 22)

GET/api/document/search-remote?patientMbo={mbo}
Nema JSON body-ja

Slanje nacrta nalaza (TC 18)

Šaljete sadržaj nalaza, zajedno s ID-jem posjete (iz TC12) i ID-jem slučaja (iz TC16 ili dohvaćenim sa CEZIH-a). Dobivate `documentOid`.

POST/api/document/send

Primjer Requesta

{ "patientMbo": "999999423", "visitId": "cmmqr24hp0...", "caseId": "cmmz82x...", "type": "011", "diagnosisCode": "Z00.0", "anamnesis": "Redovni pregled...", "finding": "Fizikalni status uredan." }

Završni korak: UI Widget za potpisivanje

Ovo je vaša najveća ušteda vremena. Ne morate implementirati rad s pametnim karticama, C++ DLL bibliotekama niti Certilia polling sustavom. Samo otvorite naš gotovi UI widget.

Kako integrirati:

  1. Kada pozovete /api/document/send dobit ćete documentOid.
  2. U vašoj G9 aplikaciji otvorite Iframe ili Popup prozor na URL:
    http://localhost:3010/sign-widget.html?oid={documentOid}
  3. Liječnik u widgetu upiše PIN pametne kartice ili odobri na mobitelu. Widget sve rješava u pozadini s CEZIH-om.
  4. Widget šalje window.postMessage vašoj aplikaciji kada je CEZIH prihvatio dokument kako biste zatvorili prozor.

Storniranje nalaza (TC 20)

Ako je dokument poslan greškom, Middleware ga može poništiti na CEZIH-u slanjem samo njegovog OID-a.

POST/api/document/cancel
{ "documentOid": "urn:oid:2.16.840.1..." }
